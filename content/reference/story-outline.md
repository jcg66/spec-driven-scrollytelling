# Story Outline

This file is reference-only Markdown.

It captures the story shape that should guide published pages without becoming a route automatically.

## Current Story Shape

- Spark: the request arrives and the system of action is triggered.
- Deconstruction: the planner-grounder split turns intent into an ordered workflow.
- Digital Eye: visual grounding reads the interface and identifies what is actionable.
- Execution Loop: the agent acts, observes feedback, and corrects until the result converges.
- Outcome: the work resolves into a clear, calm finish.

## Boundary Notes

- Keep this file as authoring support unless a future spec promotes part of it into a published page.
- Use it to align the homepage and support pages, not to create a separate route.
